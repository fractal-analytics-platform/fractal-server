199class SlurmConfig(BaseModel, extra=Extra.forbid): 

200 """ 

201 Abstraction for SLURM parameters 

202 

203 **NOTE**: `SlurmConfig` objects are created internally in `fractal-server`,

206 be part of the `FRACTAL_SLURM_CONFIG_FILE` JSON file (details on the 

207 expected file content are defined in 

208 [`SlurmConfigFile`](./#fractal_server.app.runner._slurm._slurm_config.SlurmConfigFile)). 

209 

210 Part of the attributes map directly to some of the SLURM attribues (see 

211 https://slurm.schedmd.com/sbatch.html), e.g. `partition`. Other attributes 

212 are metaparameters which are needed in fractal-server to combine multiple 

213 tasks in the same SLURM job (e.g. `parallel_tasks_per_job` or 

214 `max_num_jobs`). 

215 

216 Attributes: 

217 partition: Corresponds to SLURM option. 

218 cpus_per_task: Corresponds to SLURM option. 

219 mem_per_task_MB: Corresponds to `mem` SLURM option. 

220 job_name: Corresponds to `name` SLURM option. 

221 constraint: Corresponds to SLURM option. 

222 gres: Corresponds to SLURM option. 

223 account: Corresponds to SLURM option. 

224 gpus: Corresponds to SLURM option. 

225 time: Corresponds to SLURM option (WARNING: not fully supported). 

226 prefix: Prefix of configuration lines in SLURM submission scripts. 

227 shebang_line: Shebang line for SLURM submission scripts. 

228 extra_lines: Additional lines to include in SLURM submission scripts. 

229 tasks_per_job: Number of tasks for each SLURM job. 

230 parallel_tasks_per_job: Number of tasks to run in parallel for 

231 each SLURM job. 

232 target_cpus_per_job: Optimal number of CPUs to be requested in each 

233 SLURM job. 

234 max_cpus_per_job: Maximum number of CPUs that can be requested in each 

235 SLURM job. 

236 target_mem_per_job: Optimal amount of memory (in MB) to be requested in 

237 each SLURM job.

243 Key-value pairs to be included as `export`-ed variables in SLURM
